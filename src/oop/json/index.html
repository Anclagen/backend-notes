Introduction

JSON stands for JavaScript Object Notation. It is a lightweight data-interchange format that is easy for humans to read and write and easy for machines to parse and generate.

JSON Data Types

  • String: a sequence of characters, like "Hello World".
  • Number: an integer or a floating-point number, like 42 or 3.14.
  • Boolean: true or false.
  • Array: an ordered list of values, like ["apple", "banana", "cherry"].
  • Object: a collection of key/value pairs, like {"name": "John", "age": 30}.
  • Null: an empty value, like null.

Example json object:


        {
          "name": "John",
          "age": 30,
          "job": null,
          "isMarried": false,
          "hobbies": ["reading", "swimming", "coding"],
          "address": {
            "street": "123 Main St",
            "city": "New York",
            "state": "NY"
          }
        }

JSON Syntax Rules

  • JSON data is written as name/value pairs with property names written within double quotations, as are string values.
  • JSON data is separated by commas.
  • Curly braces hold objects.
  • Square brackets hold arrays.
  • No trailing decimals in number values, for example, “2.” is wrong.
  • No leading zeros in number values.
  • No trailing commas, like in JavaScript, we can always put a comma at the end of the last property, for example: 
    
            {
              name: "John",
              age: 30,
              job: null,
            }

JSON Processing

JSON data can be parsed and converted into JavaScript objects using the JSON.parse() method. The JSON.stringify() method can be used to convert JavaScript objects into JSON strings.

JSON Example


          const person = {
            "name": "John",
            "age": 30,
            "job": null,
            "isMarried": false,
            "hobbies": ["reading", "swimming", "coding"],
            "address": {
              "street": "123 Main St",
              "city": "New York",
              "state": "NY"
            }
          };

          const personJSON = JSON.stringify(person);
          console.log(personJSON);

          const personObject = JSON.parse(personJSON);
          console.log(personObject);

Output:


          {"name":"John","age":30,"job":null,"isMarried":false,"hobbies":["reading","swimming","coding"],"address":{"street":"123 Main St","city":"New York","state":"NY"}}
          {name: "John", age: 30, job: null, isMarried: false, hobbies: Array(3), address: {…}}

JSON.stringify()

JSON.stringify() is used to convert JavaScript objects into JSON strings. It can take three parameters:

  • Value: the JavaScript object to convert into a JSON string.
  • Replacer: a function that alters the behavior of the stringification process, or an array of String and Number objects that serve as a whitelist for selecting the properties of the value object to be included in the JSON string.
  • Space: a String or Number object that's used to insert white space into the output JSON string for readability purposes.

Example:


        const person = {
          name: 'John',
          age: 30,
          password: 'secret'
        };
        
        // Remove the password property from the JSON string
        const json = JSON.stringify(person, (key, value) => {
          if (key === 'password') return undefined;
          return value;
        });
        
        console.log(json); // {"name":"John","age":30}

JSON.parse()

JSON.parse() is used to convert JSON strings into JavaScript objects. It can take two parameters:

  • Text: the JSON string to parse.
  • Reviver: a function that transforms the results. This function is called for each member of the object.

Example:


        const json = '{"date":"2022-03-14"}';
        const dateString = JSON.parse(json);
        const dateObject = JSON.parse(json, (key, value) => {
          if (key === "date") return new Date(value);
          return value;
        });
        
        console.log(dateString.date instanceof Date); // false
        console.log(dateObject.date instanceof Date); // true

JSON with Fetch API

Fetch API is used to make network requests. It returns a Promise that resolves to the Response to that request, whether it is successful or not. You can also use it to fetch JSON data from a server.

Example:


        fetch('https://jsonplaceholder.typicode.com/posts/1')
          .then(response => response.json())
          .then(data => console.log(data));

JSON with Local Storage

Local Storage is used to store data in the browser. You can store JSON data in local storage by converting it to a string using JSON.stringify().

Example:


        const person = {
          name: 'John',
          age: 30
        };
        
        localStorage.setItem('person', JSON.stringify(person));
        
        const personJSON = localStorage.getItem('person');
        const personObject = JSON.parse(personJSON);
        
        console.log(personObject); // {name: "John", age: 30}

JSON Schema

JSON Schema is a vocabulary that allows you to annotate and validate JSON documents. It is used to define the structure of JSON data for validation, documentation, and interaction control.

  • The $schema keyword defines the standard in which the schema is written and is linked to the official JSON Schema website.
  • The $id keyword is a unique URI.
  • The title and description keywords are annotation keywords that are descriptive only. They are here to help human users understand the data.
  • The type keyword is used for validation and defines the first constraint on our JSON data.
  • The required keyword is used to define the required properties of an object.
  • The properties keyword is used to define the properties of an object.

Example:


        {
          "$schema": "http://json-schema.org/draft-07/schema#",
          "$id": "https://example.com/person.schema.json",
          "title": "Person",
          "type": "object",
          "properties": {
            "name": {
              "type": "string"
            },
            "age": {
              "type": "number"
            }
            "date": {
              "type": "string",
              "format": "date"
            }
            "interests": {
              "type": "array",
              "items": {
                "type": "string"
              }
            }
            "address": {
              "type": "object",
              "properties": {
                "street": {
                  "type": "string"
                },
                "city": {
                  "type": "string"
                },
                "county": {
                  "type": "string"
                }
                "post code": {
                  "type": "string"
                }
              },
              "required": ["street", "city", "county", "post code"]
            }
          },
          "required": ["name", "age", "date", "interests", "address"]
        }

JSON Validation

JSON Schema can be used to validate JSON data. There are various tools available to validate JSON data against a JSON Schema. Some of the popular tools are:

  • JSON Schema Validator
  • JSON Schema Lint
  • JSON Schema Test
  • JSON Schema Online

JSON-LD

JSON-LD (JavaScript Object Notation for Linked Data) is a lightweight Linked Data format. It is easy for humans to read and write and easy for machines to parse and generate. JSON-LD is designed around the concept of a "context" to provide additional meaning to the data.

Example:


        {
          "@context": "http://schema.org/",
          "@type": "Person",
          "name": "John Doe",
          "jobTitle": "Software Developer",
          "address": {
            "@type": "PostalAddress",
            "streetAddress": "123 Main St",
            "addressLocality": "New York",
            "addressRegion": "NY",
            "postalCode": "10001"
          }
        }
